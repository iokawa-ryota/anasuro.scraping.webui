# 今後のアップデート展望

## 最終目標

エンドユーザーが迷わず運用できるWebアプリにする。

## 必須要件

1. エンドユーザーが触るファイルと、内部ファイルを明確に分離する。  
2. エンドユーザーは、通常運用時にファイルを直接編集しない。  
3. 店舗取得などの設定は、Webアプリ画面から完結できるようにする。  
4. 長時間処理の進捗を `%` で表示する。  
   - スクレイピング実行時  
   - オフライン整形実行時

## 運用ルール

- エンドユーザー向け入口はルート直下の起動ファイルのみとする。  
- 設定変更は `store_list.csv` 直編集を最小化し、段階的にWeb UI経由へ移行する。  
- ログ/進捗情報は `runtime/` に集約する。

## 受け入れ条件（Doneの定義）

1. 新規ユーザーがファイル編集なしで起動し、Web UIだけで店舗設定を変更できる。  
2. スクレイピング実行中に、画面上で `0%` から `100%` まで進捗が表示される。  
3. オフライン整形中に、画面上で `0%` から `100%` まで進捗が表示される。  
4. `STARTUP_GUIDE.md` に「ユーザーが触るのはどこか」が1分で分かる説明がある。
